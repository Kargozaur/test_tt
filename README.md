Для запуска приложения достаточно иметь установленный докер-демон. Образы скачаются автоматически(
если нет, то смотрите на версии для постгреса и редиса из docker-compose. Образы внутри python докер файлов для
python смотрите внутри Docker.*(в зависимости от сервиса)).
Опционально, можно запустить через make up(является шорткатом в данном случае, если у вас есть make).

Библиотеки были выбраны те же самые, что и указанные в тестовом.
Для JWT использовалась pyjwt библиотека. Из альтернатив присутствует jose(в данном контексте
они делают ту же самую работу, поэтому какая из них была выбрана не имеет значения).
К сожалению, ввиду жизненной ситуации времени на написание тестов нет. 

Структура моно репозитория с shared директорией для обеих сервисов. Она копируется внутрь обеих образов.
В БД пишет core сервис.
Воркер беред задачи из Redis.

В проекте были опущены большинство моментов, связанных с безопасностью ввиду отсутствия времени. 
Вся sensetive информация должна находиться в *.env файл(ах)е (Например линк на postgres, redis и тп.).
Единственное, что требуется для работы приложение - это JWT_KEY внутрь .env файла. Для генерации ключа:

`openssl rand -32 hex`

Длина ключа может меняться. Т.к ни один сервис не создает токен, то нужно использовать тот же ключ, которым
подписываются токены.

При отправлении запроса, внутри хедеров обязательно должен быть указан "Authorization": "Bearer *token*"
`curl -H '"Authorization: "Bearer ..."'`

Все эндпоинты защищены через middleware. Но в FastAPI есть проблема с отсутствием возможности повесить
middleware на конкретный роутер или эндпоинт без ментальной гимнастики, поэтому все эндпоинты не из списка
внутри (файла)[shared/middleware/token_middleware.py] недоступны если у пользователя отсутствует Authorization: Bearer 
внутри хедера.
